## 1. 安装与启动

### 1.1 环境要求
- Python 3.11+
- Node.js 18+
- pnpm

### 1.2 安装依赖

```bash
cd narrative_os
pip install -e ".[dev]"
```

### 1.3 环境配置

在项目根目录创建 `.narrative_os.env`：

```env
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-ant-...
OLLAMA_BASE_URL=http://localhost:11434
DAILY_TOKEN_BUDGET=100000
CHROMA_DB_PATH=.narrative_db
LOG_LEVEL=INFO
STATE_DIR=.narrative_state
```

### 1.4 启动方式

```bash
# 一键启动（推荐）
narrative dev

# 仅后端
narrative dev --api-only

# 仅前端
narrative dev --frontend-only
```

默认地址：
- API：`http://127.0.0.1:8000`
- Frontend：`http://127.0.0.1:5173`

## 2. 功能总览

### 2.1 核心创作能力
- 章节规划：自动输出章节骨架、节点、钩子建议
- 章节生成：Planner -> Writer -> Critic -> Editor 全链路
- 快速写作：跳过 Planner 直接生成草稿
- 一致性检查：情节/角色/世界规则核验
- 去 AI 痕迹：对文本进行人味化改写

### 2.2 项目管理能力
- 项目创建、编辑、归档、软删除
- 项目级路由隔离（`/project/:id/*`）
- 章节管理与全书导出
- 全局设置与项目设置覆盖
- LLM 提供商配置与连通性测试

### 2.3 世界构建能力（沙盘版）
- 世界元信息管理（类型、主题、时代、规则）
- 地区节点：创建、详情编辑、删除
- 势力节点：创建、详情编辑、删除
- 力量体系：创建、详情编辑、删除
- 关系边：连线创建、详情编辑、删除
- 批量关系编辑
- 图谱/地图/分层/星图多视图切换 + 主题切换
- 图谱模式支持双向连线拖拽与自动布局
- 自动布局后自动 FitView
- 时间线事件新增、浏览、删除
- 势力关系映射支持名称化编辑与分值滑杆
- AI 建议关系按实体名称展示，便于理解关系指向
- 逻辑校验面板支持规则检查与 AI 深度分析
- 力量体系保存时可选同步到全局继承节点
- Finalize 写入知识库

### 2.4 角色矩阵能力
- **档案 Tab**：角色基础信息编辑（名称、特质、目标、背景、外貌描述、性格核心、别名称号），支持创建与删除
- **状态 Tab**：ECharts 六维雷达图（生命值/关系均值/情绪/记忆深度/约束数/弧光进度）+ 弧光阶段进度
- **限制 Tab**：行为约束规则列表（规则/严重度/上下文），支持新增删除，内置实时约束测试面板（输入文本 → 关键词匹配 → 显示通过/违反）
- **关系 Tab**：D3 力导向关系图（颜色编码好感度），支持新增/删除关系、拖拽节点、列表视图编辑
- **对话口吻 Tab**：VoiceFingerprint 五维编辑（高压/撒谎/回避/情绪高峰/默认篇幅）+ 语言风格 + 口头禅 + Few-Shot 对话示例（最多 10 条）+ 口吻试戏面板（AI 生成角色化对话，可保存为示例）
- **动机冲突 Tab**：Motivation 列表编辑（渴望/恐惧/张力/备注）+ 张力滑杆（颜色标注高中低）+ 张力总览（均值计算 + 建议文字）+ 场景语境与系统指令编辑
- **Pipeline 集成**：角色口吻与动机冲突自动注入 Writer Agent 提示词（Author's Note 风格），Planner 获得动机驱动规划参考，Critic 增加口吻 OOC 检测，Maintenance 自动宣泄高张力

### 2.5 TRPG 互动能力
- 会话创建、行动推进、帮回、回滚、结束
- WebSocket 流式 DM 输出
- 密度模式控制（dense/normal/sparse）

### 2.6 可视化面板
- 剧情画布、角色矩阵、记忆系统
- 质量仪表盘、风格控制台、成本统计
- Agent 工坊、插件市场、设置中心

2. 点击新建项目

## 4. CLI 使用指南

### 4.1 命令总表

| 命令 | 说明 |
|---|---|
| `narrative run` | 完整生成一章 |
| `narrative plan` | 仅生成章节规划 |
| `narrative write` | 快速生成草稿 |
| `narrative init` | 交互式初始化项目 |
| `narrative status` | 查看项目状态 |
| `narrative rollback` | 回滚章节状态 |
| `narrative cost` | 查看 token 消耗 |
| `narrative metrics` | 读取草稿并计算指标 |
| `narrative check` | 一致性检查 |
| `narrative humanize` | 去 AI 痕迹改写 |
| `narrative interactive` | TRPG 终端模式 |
| `narrative dev` | 一键启动前后端开发服务 |

### 4.2 示例

```bash
# 初始化项目
narrative init --project my_novel

# 章节规划
narrative plan --chapter 1 --summary "主角踏入禁区"

# 完整生成
narrative run --chapter 1 --summary "主角踏入禁区" --project my_novel

# 快速草稿
narrative write --chapter 1 --summary "主角踏入禁区" --project my_novel

# 一致性检查
narrative check --chapter 1 --draft ./chapter1.md

# 去 AI 痕迹
narrative humanize --input ./chapter1.md --output ./chapter1_human.md

# TRPG 终端互动
narrative interactive --project my_novel --density normal
```

## 6. 世界构建沙盘详细说明

### 6.1 模块定位
世界构建页面是“项目世界观结构化编辑器”，用于将概念文本落成可维护、可关系建模、可写回知识库的结构化世界数据。

### 6.2 主要对象
- 地区（Region）
- 势力（Faction）
- 力量体系（PowerSystem）
- 关系（Relation）

### 6.3 核心操作

#### 节点创建
1. 在画布点击新建节点
2. 选择类型（地区/势力/力量体系）
3. 在弹窗填写名称和详细信息
4. 保存后出现在画布与侧栏

#### 节点详情编辑
1. 选中节点
2. 右侧面板加载详情
3. 修改字段后保存

#### 关系管理
1. 从源节点拖拽连线到目标节点创建关系
2. 点击关系边查看关系详情
3. 可编辑关系类型、强度、描述与历史事件
4. 势力关系映射支持按名称编辑，使用滑杆设置关系分值（-1 到 1）
5. AI 建议关系以“名称 -> 名称”展示，可一键采纳
6. 支持批量关系编辑

#### 视图与时间线
1. 顶部可切换图谱/地图/分层/星图视图，便于不同维度审阅世界结构
2. 图谱视图支持自动布局，执行后会自动 FitView 到全局
3. 底部时间线支持收起/展开，并可新增或删除事件节点

#### 逻辑校验
1. 点击“逻辑校验”打开校验抽屉
2. 先查看规则校验结果（孤立节点、引用异常、体系冲突等）
3. 按需执行“AI 深度分析”获取补充问题提示

#### 力量体系同步（新增）
保存力量体系时可勾选“保存后同步到全局继承节点”：
- 对 `inherits_global=true` 的地区，自动同步专属体系
- 对未绑定体系的势力，自动绑定当前体系

#### Finalize
点击完成世界设定后，系统将世界结构写入知识库，供后续写作与一致性检查使用。

### 6.4 建议流程
1. 先定义世界 meta（类型、主题、时代、规则）
2. 再建地区和势力节点
3. 再补力量体系
4. 最后构建关系网并批量校正
5. 执行 Finalize

## 7. 推荐工作流

### 7.1 标准创作流程
1. 创建项目（`/projects`）
2. 故事概念（`/project/:id/concept`）
3. 世界构建（`/project/:id/worldbuilder`）
4. 剧情与角色梳理（`/project/:id/plot`、`/project/:id/characters`）
5. 章节写作（`/project/:id/write`）
6. 检查与润色（`/project/:id/check`、`/project/:id/humanize`）
7. 指标复盘（`/project/:id/metrics`）
8. 章节管理与导出（`/project/:id/chapters`）

### 7.2 TRPG 驱动流程
1. 先完成概念与基础世界构建
2. 在 `/project/:id/trpg` 创建会话推进剧情
3. 关键节点使用帮回/回滚控制质量
4. 结束会话后回到章节管理与质量面板复盘

## 8. 常见问题

### Q1：`narrative dev` 启动失败怎么办？
排查顺序：
1. 端口是否占用（8000/5173）
2. Python 环境是否已安装项目依赖
3. 前端依赖是否完整（`pnpm install`）

### Q2：世界构建里关系点了没详情怎么办？
请确认选中的是关系边而不是空白画布；关系详情会在右侧面板展示。若详情为空，先检查该关系是否已成功保存。

### Q3：如何批量统一力量体系？
在力量体系编辑面板保存时，勾选“保存后同步到全局继承节点”，可自动同步地区与未绑定势力。

### Q4：如何确认当前功能是否和文档一致？
以以下文件为准：
- 接口：`narrative_os/interface/api.py`
- CLI：`narrative_os/interface/cli.py`
- 路由：`frontend/src/router/index.ts`
