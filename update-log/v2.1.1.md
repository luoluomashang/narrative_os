# Narrative OS v2.1.1 更新日志

- 发布日期: 2026-04-12
- 版本基线: v2.1.1

## 本次浏览器验收补丁

- 修复缺失项目路由的前端兜底链路：访问不存在的 `/project/:id` 时，页面会给出可读错误提示并自动回退到项目列表
- 修复全局 API 错误提示把对象直接渲染成 `[object Object]` 的问题，统一解析 `detail/message/msg` 多层错误结构
- 统一 Character Matrix 与 Plot Canvas 的页头视觉层级，使其与项目主页、Writing Workbench 保持一致
- Character Matrix 进入页面后默认选中首个角色，避免首屏空白占位影响编辑效率
- 修复 Writing Workbench 规划结果展示，将原始响应改为结构化摘要，直接显示剧情骨架、节点、Hook 建议与张力曲线
- 修复 TRPG 会话着陆摘要混入 `[选项 A/B/C]` 的问题，后端归档输出改为剥离决策选项后再生成摘要与章节预览
- 修复旧运行时 SQLite 库缺少 benchmark / author skill 列导致 `POST /projects/{project_id}/benchmark/jobs` 返回 500 的问题，应用启动时会自动补齐兼容列
- 补齐 `Benchmark Studio` 在侧边栏“质量工具”中的可发现性，用户不再需要手动输入 `/project/:id/benchmark`
- 修复作者蒸馏前端校验，少于 3 部作品时提交按钮保持禁用，并提示当前已填写数量，避免与后端最小样本约束冲突
- Writing Workbench 补充 active author skill 展示区，已应用 Skill 的 scene hints / anti-rules 会与 benchmark 一起出现在写作页
- 修复 Writing Workbench 在 `WorldState` 未发布时仍允许发起章节生成的问题；现在会按前置检查直接禁用“开始生成”按钮
