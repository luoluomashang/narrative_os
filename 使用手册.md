> 版本基线：v2.1.1

本手册基于当前代码实现整理，覆盖 CLI、REST API、Web UI、世界构建沙盘、TRPG 互动、世界发布、SL 存档与 CanonCommit 正史提交。

## v2.1.1 版本说明（本次收口）

- 统一 `narrative_os.__version__` 为后端版本真相源，`/health`、包元数据与前端包版本同步为 2.1.1
- Legacy API 模块降为兼容入口，当前代码库仅维护一套主 API 定义
- 前端 OpenAPI 生成脚本已改为跨平台执行链路，移除 Windows 专用 Python 路径

- `Benchmark Studio` 已加入侧边栏“质量工具”，作者蒸馏在少于 3 部作品时会前端禁用提交按钮，并显示当前已填写数量

## 1. 安装与启动

### 1.1 环境要求
- Python 3.11+
- Node.js 18+
- pnpm

### 1.2 安装依赖

```bash
cd narrative_os
pip install -e ".[dev]"
```

### 1.3 环境配置

在项目根目录创建 `.narrative_os.env`：

```env
OPENAI_API_KEY=sk-...
ANTHROPIC_API_KEY=sk-ant-...
OLLAMA_BASE_URL=http://localhost:11434
DAILY_TOKEN_BUDGET=100000
CHROMA_DB_PATH=.narrative_db
LOG_LEVEL=INFO
STATE_DIR=.narrative_state
```

### 1.4 启动方式

```bash
# 一键启动（推荐）
narrative dev

# 仅后端
narrative dev --api-only

# 仅前端
narrative dev --frontend-only
```

默认地址：
- API：`http://127.0.0.1:8000`
- Frontend：`http://127.0.0.1:5173`

## 2. 功能总览

### 2.1 核心创作能力
- 章节规划：自动输出章节骨架、节点、钩子建议
- 章节生成：Planner -> Writer -> Critic -> Editor 全链路
- 快速写作：跳过 Planner 直接生成草稿
- 一致性检查：情节/角色/世界规则核验
- 去 AI 痕迹：对文本进行人味化改写

### 2.2 项目管理能力
- 项目创建、编辑、归档、软删除
- 项目级路由隔离（`/project/:id/*`）
- 章节管理与全书导出
- 全局设置与项目设置覆盖
- LLM 提供商配置与连通性测试

### 2.3 世界构建能力（沙盘版）
- 世界元信息管理（类型、主题、时代、规则）
- 地区节点：创建、详情编辑、删除
- 势力节点：创建、详情编辑、删除
- 力量体系：创建、详情编辑、删除
- 关系边：连线创建、详情编辑、删除
- 批量关系编辑
- 图谱/地图/分层/星图多视图切换 + 主题切换
- 图谱模式支持双向连线拖拽与自动布局
- 自动布局后自动 FitView
- 时间线事件新增、浏览、删除
- 势力关系映射支持名称化编辑与分值滑杆
- AI 建议关系按实体名称展示，便于理解关系指向
- 逻辑校验面板支持规则检查与 AI 深度分析
- 力量体系保存时可选同步到全局继承节点
- Finalize 写入知识库

### 2.4 角色矩阵能力
- **档案 Tab**：角色基础信息编辑（名称、特质、目标、背景、外貌描述、性格核心、别名称号），支持创建与删除
- **状态 Tab**：ECharts 六维雷达图（生命值/关系均值/情绪/记忆深度/约束数/弧光进度）+ 弧光阶段进度
- **限制 Tab**：行为约束规则列表（规则/严重度/上下文），支持新增删除，内置实时约束测试面板（输入文本 → 关键词匹配 → 显示通过/违反）
- **关系 Tab**：D3 力导向关系图（颜色编码好感度），支持新增/删除关系、拖拽节点、列表视图编辑
- **对话口吻 Tab**：VoiceFingerprint 五维编辑（高压/撒谎/回避/情绪高峰/默认篇幅）+ 语言风格 + 口头禅 + Few-Shot 对话示例（最多 10 条）+ 口吻试戏面板（AI 生成角色化对话，可保存为示例）
- **动机冲突 Tab**：Motivation 列表编辑（渴望/恐惧/张力/备注）+ 张力滑杆（颜色标注高中低）+ 张力总览（均值计算 + 建议文字）+ 场景语境与系统指令编辑
- **Pipeline 集成**：角色口吻与动机冲突自动注入 Writer Agent 提示词（Author's Note 风格），Planner 获得动机驱动规划参考，Critic 增加口吻 OOC 检测，Maintenance 自动宣泄高张力

### 2.5 TRPG 互动能力
- 会话创建、行动推进、帮回、回滚、结束
- WebSocket 流式 DM 输出
- 密度模式控制（dense/normal/sparse）
- 首回合支持直接点击决策选项；若返回文本内嵌选项，页面会自动提取并渲染按钮

### 2.6 可视化面板
- 剧情画布、角色矩阵、记忆系统
- 质量仪表盘、风格控制台、成本统计
- Agent 工坊、插件市场、设置中心

### 3.2 常用操作路径

#### 新建项目
1. 访问 `/projects`
2. 点击新建项目
3. 填写标题、类型、简介
4. 进入 `/project/:id`

#### 生成章节
1. 进入 `/project/:id/write`
2. 查看前置检查，确认 WorldState、角色 Drive、卷目标是否就绪
3. `WorldState` 摘要区会显示势力 / 地区 / 规则名称摘要，地区不再显示运行态 UUID
4. 如首页仍显示“未配置卷目标”，先进入 `/project/:id/plot`，在右侧填写“当前卷目标”并保存
5. 若已在 `/project/:id/benchmark` 激活 benchmark 或应用 author skill，写作页会在页头和表单区显示当前对标、scene hints 与 anti-rules
6. 若存在 error 级前置检查，`开始生成` 会保持禁用；若只有 warning，可勾选“强制生成”后继续
7. 填写章节目标摘要并选择章节号、目标字数
8. 可先点“仅规划”，此时按钮会进入“规划中…”状态，规划结果会显示在右侧输出区
9. 如需跳过 warning 类前置项，可勾选“强制生成”
10. 点击生成，观察底部 5 步 AgentRun 进度与 token 统计
10. 生成后可在 `/project/:id/trace` 查看 Run / Step / Artifact 回放，也可在 `/project/:id/agents` 以 Agent 视角查看同一条执行链路

#### Benchmark / 作者蒸馏
1. 进入 `/project/:id/benchmark`
2. 在“项目级对标”模式下粘贴参考文本，点击“生成 Benchmark Profile”
3. 生成完成后检查 `Stable Traits`、`Conditional Traits`、`Anti Traits` 与 `Scene Anchors`，确认 snippets 命中正常
4. 点击“激活当前 Profile”，Writing Workbench 会立即读取新的 benchmark 摘要
5. 切换到“作者蒸馏 Skill”，填写作者名或 corpus_group，并至少提供 3 部作品
6. 生成 Author Skill 后，在右侧查看 rules、scene patterns 与 hook 模式
7. 选择 `guide / hybrid / strict` 任一模式后应用到当前项目，Writing Workbench 会同步显示当前 skill、scene hints 与 anti-rules

#### 质量检查
1. 进入 `/project/:id/check` 运行一致性检查
2. 进入 `/project/:id/humanize` 做人味化改写
3. 在 `/project/:id/metrics` 查看评分趋势
4. 如有待提交变更，先在项目首页或 Changeset API 中审批 `CANON_PENDING` 记录

#### TRPG 会话
1. 进入 `/project/:id/trpg`
2. 创建会话，首回合会直接展示可点击选项按钮
3. 可点击选项按钮或输入自由行动持续推进
4. 需要时使用回滚/帮回
5. 结束并查看摘要

## 4. CLI 使用指南

### 4.1 命令总表

| 命令 | 说明 |
|---|---|
| `narrative run` | 完整生成一章 |
| `narrative plan` | 仅生成章节规划 |
| `narrative write` | 快速生成草稿 |
| `narrative init` | 交互式初始化项目 |

| `narrative cost` | 查看 token 消耗 |
| `narrative metrics` | 读取草稿并计算指标 |
| `narrative check` | 一致性检查 |
| `narrative humanize` | 去 AI 痕迹改写 |
| `narrative interactive` | TRPG 终端模式 |
| `narrative dev` | 一键启动前后端开发服务 |

### 4.2 示例

```bash
# 初始化项目
narrative init --project my_novel

# 章节规划
narrative plan --chapter 1 --summary "主角踏入禁区"

# 完整生成
narrative run --chapter 1 --summary "主角踏入禁区" --project my_novel

# 快速草稿
narrative write --chapter 1 --summary "主角踏入禁区" --project my_novel

# 一致性检查
narrative check --chapter 1 --draft ./chapter1.md

# 去 AI 痕迹
narrative humanize --input ./chapter1.md --output ./chapter1_human.md

# TRPG 终端互动
narrative interactive --project my_novel --density normal
```

## 6. 世界构建沙盘详细说明

### 6.1 模块定位
世界构建页面是“项目世界观结构化编辑器”，用于将概念文本落成可维护、可关系建模、可写回知识库的结构化世界数据。

### 6.2 主要对象
- 地区（Region）
- 势力（Faction）
- 力量体系（PowerSystem）
- 关系（Relation）

### 6.3 核心操作

#### 节点创建
1. 在画布点击新建节点
2. 选择类型（地区/势力/力量体系）
3. 在弹窗填写名称和详细信息
4. 保存后出现在画布与侧栏

#### 节点详情编辑
1. 选中节点
2. 右侧面板加载详情
3. 修改字段后保存

#### 关系管理
1. 从源节点拖拽连线到目标节点创建关系
2. 点击关系边查看关系详情
3. 可编辑关系类型、强度、描述与历史事件
4. 势力关系映射支持按名称编辑，使用滑杆设置关系分值（-1 到 1）
5. AI 建议关系以“名称 -> 名称”展示，可一键采纳
6. 支持批量关系编辑

#### 发布动作
1. 点击“完成世界设定”会把当前结构化世界数据写入知识库
2. 点击“发布运行态”会调用 `POST /projects/{project_id}/world/publish`，生成 `RuntimeWorldState`
3. 发布成功后，项目首页与 Writing Workbench 会立即显示“世界已发布”

### 6.4 世界书能力（当前实现）

当前版本不单独提供“世界书模块”页面，世界书能力通过以下链路实现：

1. 在世界构建沙盘维护结构化世界数据（地区/势力/力量体系/关系/时间线）。
2. 执行 `POST /projects/{project_id}/world/finalize`，将世界数据写入 `knowledge_base.json`。
3. 使用 `POST /projects/{project_id}/world/publish` 将沙盘编译为可运行的 `WorldState`。

### 6.5 世界书最佳实践（推荐）

1. 概念先行：先填写故事概念，再进入世界构建，减少后续重构。
2. 关系优先：先建立关键地区/势力关系，再细化描述字段。
3. 每轮阶段性完善后执行一次 Finalize，保证知识库同步。
4. 写作或 TRPG 前务必执行一次 Publish，确保 `RuntimeWorldState` 已更新。
5. 写作前先做一次逻辑校验，避免无效节点和断裂关系进入正文。
6. 写作后结合一致性检查和人味化，回补世界设定缺口。

#### 视图与时间线
1. 顶部可切换图谱/地图/分层/星图视图，便于不同维度审阅世界结构。
2. 图谱视图支持自动布局，执行后会自动 FitView 到全局。
3. 底部时间线支持收起/展开，并可新增或删除事件节点。

#### 逻辑校验
1. 点击“逻辑校验”打开校验抽屉。
2. 先查看规则校验结果（孤立节点、引用异常、体系冲突等）。
3. 按需执行“AI 深度分析”获取补充问题提示。

#### 力量体系同步（新增）
保存力量体系时可勾选“保存后同步到全局继承节点”：
- 对 `inherits_global=true` 的地区，自动同步专属体系。
- 对未绑定体系的势力，自动绑定当前体系。

#### Finalize
点击完成世界设定后，系统将世界结构写入知识库，供后续写作与一致性检查使用。

### 6.6 建议流程
1. 先定义世界 meta（类型、主题、时代、规则）。
2. 再建地区和势力节点。
3. 再补力量体系。
4. 最后构建关系网并批量校正。
5. 执行 Finalize。

## 7. 世界发布管线（WorldPublish）

`POST /projects/{project_id}/world/publish` 将世界沙盘编译为可运行的 `WorldState`，持久化到数据库和文件系统。

### 7.1 流程

1. **读取沙盘数据**：从 DB 读取 `WorldSandboxData`（地区、势力、规则等）
2. **WorldValidator 校验**：输出 errors / warnings / suggestions 三级报告
3. **WorldCompiler 编译**：沙盘 → `WorldState`（运行态）
4. **WorldRepository 持久化**：写入 DB + `.narrative_os/{project_id}/knowledge_base.json`
5. 返回 `status: "published"` 或 `status: "validation_failed"`

### 7.2 响应结构

```json
{
  "status": "published",
  "world_version": "v1712345678",
  "warnings": [],
  "publish_report": {
    "regions_compiled": 3,
    "factions_compiled": 2,
    "rules_compiled": 5,
    "power_systems_compiled": 1
  }
}
```

### 7.3 校验失败响应

```json
{
  "status": "validation_failed",
  "errors": ["世界名称不能为空"],
  "warnings": [],
  "suggestions": []
}
```

## 8. 互动模式与控制层（TRPG + ControlMode）

### 8.1 四档控制模式

| 模式 | 值 | 说明 |
|------|-----|------|
| 用户主驱动 | `user_driven` | 玩家完全控制主角，NPC 由 DM 自由叙述 |
| 半代理辅助 | `semi_agent` | 允许 DM 为主角提供行动建议（可拒绝） |
| 导演叙事 | `directed` | DM 主导叙事节奏，玩家做重要选择 |
| 全自动推演 | `director` | DM 推演所有角色含主角，生成连续叙事 |

### 8.2 控制模式切换

通过 `InteractiveSession.control_mode` 和 `ControlModeConfig` 切换：

```python
from narrative_os.core.interactive_modes import ControlMode, ControlModeConfig

session.control_mode = ControlMode.SEMI_AGENT
session.mode_config = ControlModeConfig(
    mode=ControlMode.SEMI_AGENT,
    allow_protagonist_proxy=True,
)
```

### 8.3 SandboxSimulator（非主角推演）

在 `SEMI_AGENT` / `DIRECTOR` 模式下，`SandboxSimulator.simulate_turn()` 推演在场非主角的行动意图，输出 `AgendaDelta` 列表。

## 9. SL 系统（存档/读档）

### 9.1 SaveStore — 存档

```python
from narrative_os.core.save_load import get_save_store

store = get_save_store(session_id)
save = store.create(
    session,
    trigger="manual",          # "manual" | "auto" | "chapter_end" 等
    plot_position="第三章·天玄城决战前",
)
print(f"存档 ID: {save.save_id}")
```

### 9.2 SoftRollback — 软读档

```python
from narrative_os.core.save_load import SoftRollback

# 获取存档点
save_point = store.get(save_id)

# 恢复 session（保留 memory_summary，注入微扰防止同路径重现）
restored_session = SoftRollback.restore(session, save_point)
```

### 9.3 REST API

```
GET  /projects/{pid}/sessions/{session_id}/saves        # 列出存档
POST /projects/{pid}/sessions/{session_id}/save         # 手动存档
POST /projects/{pid}/sessions/{session_id}/saves/{id}/restore  # 读档
```

### 9.4 存档保留策略

`SoftRollback` 读档时：
- 恢复 `history`、`turn`、`scene_pressure`
- 保留 `memory_summary_cache`（不完全失忆）
- 注入系统微扰提示，防止完全相同路径重现
- 重置 `phase → PING_PONG`，清除 `open_decision`

## 10. CanonCommit 正史提交

### 10.1 三种提交模式

| 模式 | 枚举值 | 说明 |
|------|--------|------|
| 仅会话记录 | `SESSION_ONLY` | 互动内容保留在会话历史，不进入正史 |
| 草稿章节 | `DRAFT_CHAPTER` | 生成草稿等待人工审核，ChangeTag = DRAFT |
| 正式章节 | `CANON_CHAPTER` | 二次确认后直接提交正史，ChangeTag = CANON_PENDING |

### 10.2 创建变更集

```python
from narrative_os.core.evolution import (
    CanonCommit, ChangeSource, ChangeTag, SessionCommitMode, WorldChange
)

canon = CanonCommit()
cs = canon.create_changeset(
    project_id="my_novel",
    source=ChangeSource.INTERACTIVE,
    commit_mode=SessionCommitMode.DRAFT_CHAPTER,
    draft_content="第五章草稿：林枫获悉血魔尊的真实身份……",
    changes=[
        WorldChange(
            source=ChangeSource.INTERACTIVE,
            chapter=5,
            tag=ChangeTag.DRAFT,
            change_type="timeline_event",
            description="揭露血魔尊真实身份",
            after_value={"event": "身份揭露", "chapter": 5},
        )
    ],
)
print(f"变更集 ID: {cs.changeset_id}")
```

### 10.3 审批 + 正史提交

```python
# 批量审批所有变更
canon.approve_all(cs.changeset_id)

# 二次确认后提交正史
cs.canon_confirmed = True
canon.commit_to_canon(cs.changeset_id)
# 提交后 ChangeTag 变为 CANON_CONFIRMED
```

### 10.4 REST API

```
GET  /projects/{pid}/changesets               # 列出变更集
GET  /projects/{pid}/changesets/{cs_id}       # 变更集详情
POST /projects/{pid}/changesets/{cs_id}/approve  # 批量审批
POST /projects/{pid}/changesets/{cs_id}/reject   # 驳回
POST /projects/{pid}/sessions/{session_id}/commit  # 会话结束提交
```

## 11. 推荐工作流

### 7.1 标准创作流程
1. 创建项目（`/projects`）
2. 故事概念（`/project/:id/concept`）
3. 世界构建（`/project/:id/worldbuilder`）

7. 指标复盘（`/project/:id/metrics`）
8. 章节管理与导出（`/project/:id/chapters`）

### 7.2 TRPG 驱动流程
1. 先完成概念与基础世界构建
2. 在 `/project/:id/trpg` 创建会话推进剧情
3. 关键节点使用帮回/回滚控制质量
4. 在创作环节通过记忆检索页（`/project/:id/memory-search`）召回关键设定
5. 在执行链路页（`/project/:id/trace`）排查章节级执行过程
6. 结束会话后回到章节管理与质量面板复盘

## 12. 常见问题

### Q1：`narrative dev` 启动失败怎么办？
排查顺序：
1. 端口是否占用（8000/5173）
2. Python 环境是否已安装项目依赖
3. 前端依赖是否完整（`pnpm install`）

### Q2：世界构建里关系点了没详情怎么办？
请确认选中的是关系边而不是空白画布；关系详情会在右侧面板展示。若详情为空，先检查该关系是否已成功保存。

### Q3：如何批量统一力量体系？
在力量体系编辑面板保存时，勾选“保存后同步到全局继承节点”，可自动同步地区与未绑定势力。

### Q4：如何确认当前功能是否和文档一致？
以以下文件为准：
- 接口：`narrative_os/interface/api.py`
- CLI：`narrative_os/interface/cli.py`
- 路由：`frontend/src/router/index.ts`
