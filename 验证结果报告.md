## 三、发现并修复的问题

### Bug #1 — ChromaDB EphemeralClient 全局单例污染

**严重程度：** 高（导致测试结果不可信）  
**文件：** `tests/test_core/test_memory.py`  
**表现：** 3 条测试失败：

```
FAILED tests/test_core/test_memory.py::TestArchive::test_archive_empty_mid_returns_empty
FAILED tests/test_core/test_memory.py::TestCollectionCounts::test_counts_initially_zero
FAILED tests/test_core/test_memory.py::TestCollectionCounts::test_count_after_write
```

**根本原因：**  
`chromadb.EphemeralClient()` 在整个进程中只实例化一次全局共享的内存数据库。无论在测试中创建多少个独立的 `MemorySystem` 对象，它们底层都读写同一个 SQLite in-memory 实例，导致跨测试用例的数据污染：

```python
# 验证复现：
c1 = chromadb.EphemeralClient()
c2 = chromadb.EphemeralClient()
coll1 = c1.create_collection("test")
coll1.add(documents=["hello"], ids=["1"])
coll2 = c2.get_collection("test")
print(coll2.count())  # 输出 1 — c2 能看到 c1 写入的数据！
```

**修复方案：**  
将测试 fixture 从触发 `EphemeralClient` 的 `persist_dir` 参数改为使用 `db_path` + pytest `tmp_path`，使每个测试用例获得真正隔离的磁盘目录：

```python
# 修复前（错误）：
@pytest.fixture
def mem(tmp_path) -> MemorySystem:
    return MemorySystem(project_id="test_project", persist_dir=str(tmp_path))
    # persist_dir 触发 EphemeralClient() — 全局单例，无隔离！

# 修复后（正确）：
@pytest.fixture
def mem(tmp_path) -> MemorySystem:
    return MemorySystem(project_id="test_project", db_path=str(tmp_path / "chroma"))
    # db_path 触发 PersistentClient(path=...) — pytest tmp_path 保证每次唯一目录，完全隔离
```

**复测结果：** ✅ 修复后 15/15 内存测试全部通过

该文件负责启动/停止 `uvicorn` 和 `vite` 子进程、管理 PID 文件、检测孤儿进程、处理端口冲突。其未覆盖路径均需要：

- 真实操作系统进程生命周期（`subprocess.Popen`、`psutil`）
- 实际网络端口绑定
- 跨平台信号处理（`signal.SIGTERM` / Windows taskkill）
- 多秒级等待逻辑（进程启动/停止等待）

这类基础设施代码属于"部署级集成测试"范畴，在 CI 单元测试中强制覆盖会引入不稳定因素（端口冲突、环境依赖、超时风险）。**已有独立集成测试文件 `tests/test_dev_server.py` 覆盖其可测路径，属合理安排。**

### 8.3 发现并修复的前端问题

#### Bug F-1（严重）：路由 `/project/:id/chapters` 指向错误组件

- **位置：** `src/router/index.ts`
- **现象：** 点击侧边栏"章节管理"跳转后显示全局项目列表页，而非当前项目章节
- **根因：** 路由 `component` 误配置为 `ProjectManager/index.vue`，且对应页面组件不存在
- **修复：** 新建 `pages/ChapterManager/index.vue`，实现章节列表展示、正文查看、导出功能；路由改为导入新组件
- **使用 API：** `GET /projects/{id}/chapters`、`GET /projects/{id}/chapters/{n}`、`GET /projects/{id}/export`

#### Bug F-2（中等）：MetricsDashboard 8D 雷达图数据列全部显示默认值 5

- **位置：** `src/pages/MetricsDashboard/index.vue`（第 184–187 行）
- **现象：** 8D 雷达图无论写入多少章节数据，8 个维度均显示 5（默认值）
- **根因：** 前端使用旧 key 名 `item.qd_01_catharsis`、`item.qd_02_golden_finger` 等，后端实际返回字段为 `qd_01`–`qd_08`；所有字段为 `undefined`，触发 `typeof v === 'number' ? v : 5` 的 fallback
- **附加问题：** 后端返回值为 `0.0–1.0` 浮点数，雷达图 max 为 10，未做换算
- **修复：** key 名改为 `item.qd_01`–`item.qd_08`，值乘以 10 后取整 `Math.round(v * 10)`

#### Bug F-3（中等）：PluginMarketplace 插件开关切换后本地状态不更新

- **位置：** `src/pages/PluginMarketplace/index.vue`（`togglePlugin` 函数）
- **现象：** 切换插件开关后 UI 保持乐观更新状态（由 `el-switch v-model` 驱动），但 API 调用成功后未从服务端同步 `enabled` 字段；若后端返回 `false`（限制类插件自动禁用），前端状态不一致
- **修复：** API 成功后读取 `res.data.enabled` 赋值 `plugin.enabled`；失败时保存原始值 `prev` 并回滚

#### Bug F-4（低）：api.py 使用已弃用的 HTTP 状态码常量

- **位置：** `narrative_os/interface/api.py` 第 390 行
- **现象：** `HTTP_422_UNPROCESSABLE_ENTITY` 弃用警告（FastAPI / Starlette 新版已改名）
- **修复：** 改为 `HTTP_422_UNPROCESSABLE_CONTENT`

#### Bug F-5（中等）：StyleConsole 实时预览永远显示 API 不可用

- **位置：** `src/pages/StyleConsole/index.vue`（`refreshPreview` 函数）
- **现象：** 调整任意风格滑块后，预览框始终显示"（API 不可用，无法生成实时预览）"
- **根因：** 前端调用 `POST /style/extract` 并期望响应中含 `preview` 字段，但后端 `StyleExtractResponse` 从不返回该字段（只返回 `style_params`、`confidence` 等）
- **修复：** 移除 API 调用依赖，改为本地直接渲染当前各维度参数值作为预览文本（同步操作，无需等待网络，无需修改后端）
