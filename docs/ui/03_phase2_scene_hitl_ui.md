## 2. 一致性检查与 Diff 视图补丁（Revision Flow）

**后端映射**：`agents/editor.py` → `EditorAgent.edit()`（补丁生成）

### 2.1 内联标注交互

1. CriticAgent 扫描完成后，问题段落获得底边样式（见 §1.3）
2. 用户**悬停/点击**问题高亮区 → 右栏推出"诊断详情卡"
3. 诊断详情卡包含：
   - 违反的规则条目
   - 原始参考数据（来源于 `MemorySystem` 或 `BehaviorConstraint`）
   - 操作按钮：**请求重写** / **标记忽略** / **手动修改**

### 2.2 Diff 视图（补丁对比）

用户点击"请求重写"后，EditorAgent 生成补丁，触发 Diff 展开：

```
┌── 原始文本 (红色删除线) ─────────┬── AI 补丁 (绿色高亮) ─────────┐
│  他悄悄接受了~~敌军的提议~~，     │  他悄悄接受了敌军的情报，但   │
│  内心充满矛盾。                  │  以"探听虚实"为由搪塞自己。  │
└──────────────────────────────┴──────────────────────────────┘

         逐句操作按钮（每句末尾）：
         [✓ Accept]  [✗ Reject]  [✏ Modify]
```

**规则**：
- 每个补丁块独立决策，禁止自动批量接受
- **Modify**：打开行内小型文本框允许用户在补丁基础上修改
- 接受后段落颜色恢复正常，问题标注消除
- 拒绝后保留原文，右栏显示说明"用户保留原始版本"

## 3. 人机拦截网（HITL Approval Gates）

**后端映射**：`infra/hitl.py` → `HITLManager.checkpoint()` / `.approve()` / `.reject()`

### 3.1 高风险 · 阻塞式审批模态框（Blocking Modal）

**触发条件**：`HITLManager.checkpoint(gate_type="high_risk", ...)` 被调用时

（示例：AI 试图将核心配角标记为死亡 / 永久修改重要阵营状态）

```
屏幕效果：
  中栏写作区被半透明深色遮罩覆盖（#0a0a0b · opacity 80%）

┌── HITL 审批模态框（#1b1c22 · 边框 #ff2e88 2px · 圆角 12px）──┐
│  ⚠  AI 意图声明                                              │
│  ───────────────────────────────────────────────────────   │
│  意图探测：当前推演将导致以下不可逆操作：                        │
│    • 角色 [白明远] 生命体征 → 归零                            │
│    • 世界观设定修改：[北朔阵营] 阵营属性 → 中立               │
│                                                             │
│  操作前须确认授权。所有智能体已暂停。                           │
│  ───────────────────────────────────────────────────────   │
│  [✓ 批准执行]  [↺ 重新推理]  [✗ 取消操作]  [✋ 手动接管]      │
└────────────────────────────────────────────────────────────┘
```

**四个按钮语义**：

| 按钮 | 调用后端 | 行为 |
|------|---------|------|
| 批准执行 | `HITLManager.approve(gate_id)` | 放行，继续生成 |
| 重新推理 | 触发 `/chapters/run` 重试 | 清除当前 AI 意图，重新生成 |
| 取消操作 | `HITLManager.reject(gate_id)` | 放弃本次操作，状态回滚 |
| 手动接管 | 解锁中栏文本编辑器 | 用户直接手写，临时绕过 AI |

**约束**：模态框弹出后，**所有后台 Agent 调用必须暂停**，不得静默执行任何状态变更。

### 3.2 低风险 · 非阻塞异步通知托盘（Non-Blocking Tray）

**触发条件**：`HITLManager.checkpoint(gate_type="low_risk", ...)` 被调用时

（示例：AI 完成 5 套场景草案备选，等待用户选择）

```
界面右上角：
  🔔 [审批托盘图标]  ← 角标红点显示待处理数量 "3"

点击展开托盘（右上角下拉）：
┌── 审批托盘 ─────────────────────────────────────────────┐
│  ● [草案A·环境渲染·自然照明版] [预览] [✓] [✗]         │
│  ● [草案B·环境渲染·阴暗压抑版] [预览] [✓] [✗]         │
│  ● [草案C·人物对话·激烈争论版] [预览] [✓] [✗]         │
│                                                        │
│  [批量批准全部]  [批量拒绝全部]                          │
└────────────────────────────────────────────────────────┘
```

**约束**：
- 用户未处理托盘时，**不阻断主创作流程**
- 托盘最多积压 10 条，超出时自动合并同类型待审项
- `ApprovalRequest.timeout > 0` 时，若超时自动显示 TIMEOUT 状态并从托盘移除

## 4. 后端状态流转对照

```
用户点击 [生成] 按钮
  → POST /chapters/run
      → PlannerAgent.plan()       [左栏·节点更新]
      → WriterAgent.write()       [中栏·打字机]
      → CriticAgent.critique()    [右栏·诊断卡]
          ↓ 发现高风险意图
      → HITLManager.checkpoint()  [阻塞模态框]
          ↓ 用户批准
      → EditorAgent.edit()        [Diff 视图]
          ↓ 最终接受
      → MaintenanceAgent.maintain() [章末状态更新]
      → 返回 RunChapterResponse    [状态条·完成]
```
