# Narrative OS v2.1.0 更新日志

- 发布日期: 2026-04-11
- 版本基线: v2.1.0

## 七阶段验收结论

- 创作模式 E2E 通过：从 Concept、World Publish、角色四层、Plot、Authoring Runtime 到 Canon 审批全链路可用
- 互动模式 E2E 通过：TRPG 会话、规则裁定、Agenda 推演、SL、死锁解套、三种提交模式均验证通过
- 跨模式切换 E2E 通过：TRPG 造成的世界变化、Lorebook 与角色 Runtime 能在后续写作模式中被感知
- 性能基准通过：`WorldCompiler`、`NarrativeCompiler`、`Lorebook`、`WorldChangeSet`、`MemorySystem` 等关键指标达标
- 全量测试通过：`912 passed, 1 skipped`

## 重点新增

- Writer 强制消费 `NarrativeCompiler.compile_authoring()` 编译包
- Maintenance 成为权威回写责任人，负责角色 Runtime、Plot 推进、记忆锚点、Hook 与 `CANON_PENDING` 变更集
- 新增 `Writing Workbench`、项目状态机首页与 pending changes 提示
- 新增写作上下文、Run 列表、Run 步骤树与审批接口
- Trace 持久化链路修复运行时数据库漂移问题，列表与步骤树保持同库一致

## 发布后清理

- 为 `WorldSandboxData` / `ConceptData` 增加赋值期校验，收敛 `world_type` 字符串写入导致的枚举序列化 warning
- 将章节与 TRPG 的非致命持久化失败从 `warnings.warn` 改为结构化日志，避免测试与运行时出现误导性告警
- 移除弃用的 422 常量写法，回归结果更新为：`912 passed, 1 skipped` 且无 warnings summary

## 2026-04-12 浏览器回归修复

- 修复项目首页 `Concept` 状态误判：已保存到数据库的概念现在会正确显示为“概念已初始化”
- 世界构建页新增“发布运行态”按钮，发布后项目首页与 Writing Workbench 会同步显示 `WorldState` 已发布

- 基于浏览器真实操作复核了项目创建、概念保存、世界构建、运行态发布、角色创建、Writing Workbench 与 TRPG 会话启动链路
